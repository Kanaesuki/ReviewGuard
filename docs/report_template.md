# 虚假评论/水军识别实验报告

## 1. 实验背景与任务说明

本实验模仿垃圾文本检测实践工程，面向电商/服务平台评论文本，完成虚假评论或水军评论识别任务。输入为评论文本，输出为二分类标签：正常评论或疑似虚假评论。

## 2. 数据集说明

- 数据来源：京东电商真实/虚假评论数据集
- 样本数量：9556 条（训练集 5556、验证集 2000、测试集 2000）
- 虚假评论/水军评论数量：5783 条（60.52%）
- 真实用户评论数量：3773 条（39.48%）
- 标签定义：`0`=虚假/水军评论，`1`=真实用户评论
- 数据划分方式：训练集、验证集、测试集相互独立，分别存放于 `data/train.csv`、`data/dev.csv`、`data/test.csv`

更完整的统计见 `docs/data_statistics.md`。

## 3. 数据预处理

主要步骤包括（实现见 `src/preprocess.py`）：

1. HTML 实体解码，并去除 HTML/XML 标记、URL、非中文字符和多余空格。
2. 使用 jieba 对中文评论进行分词。
3. 使用 `data/hit_stopwords.txt` 与内置词表合并去停用词，并过滤单字词。
4. 将处理后的词序列输入特征提取模块（训练时由 `train.py` 自动调用）。

预处理样例与长度分布见 `docs/data_statistics.md` 第 4、5 节。

## 4. 算法设计思路

本实验采用“文本清洗 + 分词 + TF-IDF 特征 + 机器学习分类器”的流程。TF-IDF 可以衡量词语在单条评论中的重要程度，并降低常见词的影响。分类器部分可对比逻辑回归、朴素贝叶斯和线性 SVM。

可扩展方向：

- 字符级 n-gram：捕获“返现”“冲”“闭眼入”等短语模式。
- 行为特征：账号评论频率、重复模板比例、评论时间间隔。
- 情感与评分一致性：五星评分但文本含负面词，可能存在异常。

## 5. 模型实现与代码说明

核心代码：

- `src/preprocess.py`：文本清洗、分词、停用词过滤。
- `src/train.py`：加载数据、划分训练集测试集、训练模型、输出评估指标。
- `src/predict.py`：加载模型并预测单条评论。
- `app.py`：可视化演示页面。

## 8. 技术挑战与解决方案

示例：

- 挑战 1：中文文本没有天然空格，分词质量影响模型效果。
  - 解决方案：使用 jieba 分词，并加入领域停用词。
- 挑战 2：水军评论常常伪装成正常评论。
  - 解决方案：加入 n-gram 特征，捕获模板化短语。
- 挑战 3：类别不平衡导致模型偏向多数类。
  - 解决方案：使用 `class_weight="balanced"` 或重采样。
