# 四人分工建议

## 成员 A：数据与预处理负责人

- 收集评论数据，整理为 `text,label` 格式。
- 完成数据清洗、分词、停用词过滤。
- 统计数据规模、类别比例、文本长度分布。
- 在报告中负责“数据集说明”和“数据预处理”部分。

## 成员 B：特征与模型负责人

- 实现 TF-IDF、n-gram 等文本特征。
- 训练逻辑回归、朴素贝叶斯、SVM 等模型。
- 比较不同模型和参数组合。
- 在报告中负责“算法设计思路”和“模型实现”部分。

## 成员 C：评估与结果分析负责人

- 计算 accuracy、precision、recall、F1、混淆矩阵。
- 选取预测正确和预测错误的典型案例进行分析。
- 思考为什么会误判，例如广告词、返现词、模板化表达、正常夸赞混淆。
- 在报告中负责“实验结果”和“结果分析”部分。

## 成员 D：工程展示与汇报负责人

- 维护 Streamlit 演示页面，准备视频展示流程。
- 整理课堂展示 PPT。
- 录制或剪辑展示视频，确保不只是读指标，而是展示真实输入和预测过程。
- 在报告中负责“系统演示”和“代码说明”部分。

## 推荐协作节奏

1. 第 1 天：确定数据来源和标签规则。
2. 第 2 天：完成基础工程运行和第一版模型。
3. 第 3 天：补充实验对比和错误案例。
4. 第 4 天：完善报告、PPT、视频展示。

## 课堂展示分配

- A：问题背景、数据来源、预处理流程。
- B：算法设计、特征提取、模型训练。
- C：实验结果、混淆矩阵、误判分析。
- D：系统演示、技术挑战、解决方案、总结。
